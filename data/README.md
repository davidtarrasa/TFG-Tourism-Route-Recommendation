# Data Layout

## Directory structure

- `raw/`: raw input files (not versioned — see note below)
- `processed/`: cleaned data ready for Postgres load
- `reports/`: generated evaluation outputs and figures

## Processed files

| File | Description |
|------|-------------|
| `processed/std_clean.csv` | Cleaned check-in data (user_id, trail_id, venue_id, timestamp, city_qid) |
| `processed/pois_enriched_Q35765.json` | POI metadata for Osaka |
| `processed/pois_enriched_Q406.json` | POI metadata for Istanbul |
| `processed/pois_enriched_Q864965.json` | POI metadata for Petaling Jaya |
| `processed/category_price_labels.json` | Category and price tier labels per venue |

## PostgreSQL schema (after load)

Key tables populated by `src/etl/08_load_postgres.py`:

- `visits`: check-in records (trail_id, user_id, venue_id, timestamp, city_qid)
- `pois`: POI metadata (venue_id, name, lat, lon, rating, primary_category, city_qid)
- `poi_categories`: full category taxonomy per POI (many-to-one with pois)
- `saved_routes`: user-saved itineraries (frontend persistence)

## City QID mappings

| Wikidata QID | City |
|---|---|
| Q35765 | Osaka, Japan |
| Q406 | Istanbul, Turkey |
| Q864965 | Petaling Jaya, Malaysia |

## Loading data into Postgres

```bash
docker compose up -d db pgadmin
python src/etl/08_load_postgres.py --dsn postgresql://tfg:tfgpass@localhost:55432/tfg_routes
```

Requires `processed/` to contain the files listed above.

## Reports directory

- `reports/benchmarks/`: benchmark JSON summaries and per-city eval snapshots
- `reports/eval_<qid>_latest.json`: latest ranking eval results per city (used by figure generator)
- `reports/eval_routes_<qid>_latest.json`: latest route quality eval results per city
- `reports/figures/tfg/`: 26 thesis figures generated by `scripts/generate_tfg_figures.py`
- `reports/routes/`: sample route HTML/GeoJSON exports

## Raw data note

Raw files in `raw/` are not versioned (large or restricted). Processed files in `processed/` are included so the system can start immediately without re-running the full ETL pipeline.
